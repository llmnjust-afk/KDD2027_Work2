\section{Experiments and Results}
\label{sec:exp}

\subsection{Experimental Setup}
\label{sec:setup}

\textbf{Models.} We evaluate five frontier LLMs via an OpenAI-compatible endpoint: GPT-4o, GPT-4o-mini, DeepSeek-V3 (\texttt{deepseek-chat}), DeepSeek-R1 (\texttt{deepseek-reasoner}), and Qwen3-max (\texttt{qwen3-max-2026-01-23}). All models use temperature 0 and a maximum of 6--8 ReAct steps.

\input{tables/table_task_summary}

\textbf{Protocol.} Each (model, task) pair is run for 3 repetitions. The main experiment (95 tasks $\times$ 5 models $\times$ 2 variants $\times$ 3 repeats) yields 2{,}850 runs on synthetic tasks; the UCI experiment (23 tasks $\times$ 5 models $\times$ 3 repeats) yields 345 runs; the DSBench experiment (200 tasks $\times$ 2 models $\times$ 3 repeats) yields 1{,}200 runs. In total we analyse \textbf{5{,}820 agent runs} and \textbf{3{,}781 failures}. All tables in this paper are auto-generated from a single source-of-truth manifest with assertion checks\footnote{See \texttt{build\_manifest\_and\_tables.py} in our supplementary code.}.

\textbf{Statistical tests.} Verifier comparisons use the McNemar test (appropriate for paired binary outcomes), reporting $b$ (baseline correct \& verifier wrong), $c$ (baseline wrong \& verifier correct), and the $\chi^2$ $p$-value.

\subsection{Result 1: Comprehensive Model Comparison}
\label{sec:res-models}

\input{tables/table_main_results}

\Cref{tab:main-results} presents the comprehensive results. DeepSeek-V3 achieves the highest success rate on both synthetic (88.4\%) and UCI real data (94.2\%). The corrected SFR reveals a task-dependent pattern: on synthetic trap tasks, SFR is 39--100\% (failures are often wrong answers, not crashes); on UCI real data, SFR ranges from 0\% to 100\% depending on model; on DSBench, SFR is 0--38\%. The high SFR is specific to tasks where code runs but reasoning fails, not a universal property.

\subsection{Result 2: Failure Bifurcation}
\label{sec:res-bifurcation}

\input{tables/table_null_model}

\Cref{tab:null-model} reports SFR by task suite. The SFR is 86.2\% on synthetic tasks, 3.8\% on UCI, and 18.9\% on DSBench. We note that SFR and execution-success rate share the variable $E$ (loud failures), so their correlation is partially mechanical. The $R^2 = 0.493$ is reported as a \emph{descriptive statistic}, not a causal decomposition: it cannot be interpreted as ``49\% mechanical, 51\% real.'' A proper test would require a multinomial mixed-effects model with model, task difficulty, and suite as random effects, which we leave to future work.

\subsection{Result 3: Taxonomy Validation}
\label{sec:res-taxonomy}

We validate the five-stage taxonomy via LLM-as-judge annotation (GPT-4o-mini, 100 traces, blind to the rule-based label). After fixing a classifier bug that incorrectly labeled \texttt{final\_answer=None} traces as silent, Cohen's $\kappa = 0.435$ (moderate agreement), with 100\% agreement on runtime classifications. The remaining disagreement centers on the boundary between \texttt{output\_mismatch} and \texttt{answer\_error}. We acknowledge that LLM-as-judge cannot replace human annotation; full human validation with $\geq 3$ annotators is needed for a definitive $\kappa$.

\subsection{Result 4: Zero Self-Correction}
\label{sec:res-recovery}

Across all 5{,}820 runs and all five models, the recovery rate is 0\%: no agent ever corrected a silent failure without external intervention. We note that this is partly a consequence of the experimental protocol: silent failures produce no error signal, so the agent has no opportunity to detect and retry. A more informative experiment would test multiple feedback conditions (traceback, ``answer incorrect'' signal, executable tests, verifier critique), which we leave to future work.

\subsection{Result 5: Oracle-Guided Repair}
\label{sec:res-causal}

\input{tables/table_oracle_repair}

Oracle-guided counterfactual repair succeeds on 66\% of failed tasks where ground-truth code is available (\Cref{tab:oracle-repair}). This demonstrates \emph{repairability}, not causal attribution: as the negative control in \Cref{sec:disc-negative-control} shows, injecting the oracle solution produces identical results regardless of which step is replaced, because the ground-truth code independently computes the correct answer. We therefore report oracle repair as an upper bound on reparability rather than as evidence of step-level causal localisation. A full causal analysis with necessity/sufficiency testing is future work.

\subsection{Result 6: Verifier Ablation}
\label{sec:res-verifier}

\input{tables/table_verifier}

\Cref{tab:verifier} reports the McNemar test results. The verifier significantly helps Qwen3-max ($+24.9\%$, $p<0.001$) and GPT-4o ($+19.0\%$, $p<0.001$), but is not significant for GPT-4o-mini, DeepSeek-V3, or DeepSeek-R1. Note that GPT-4o's improvement was not detected by the paired $t$-test in our earlier analysis, highlighting the importance of using the correct statistical test for binary outcomes. The verifier's benefit to Qwen3-max may partly reflect format/protocol fix-up rather than reasoning correction, since Qwen3-max's low baseline performance is partly due to ReAct parser incompatibility.
